## Система аутентификации и авторизации

Эта система позволяет пользователям проходить регистрацию, авторизацию, обновление профилей и удаление аккаунтов. Основные компоненты включают модели пользователей, роли и разрешения, определяющие доступ к разным частям приложения.

### Архитектура и схема БД

Система построена вокруг двух ключевых компонентов:

1. **Пользователи**: каждый зарегистрированный пользователь имеет уникальный идентификатор и связь с одной или несколькими ролями.
2. **Роли**: роли определяют набор действий, разрешенных для определенного ресурса.
3. **Разрешения**: каждая роль связана с набором разрешений, позволяющих читать, писать, изменять или удалять конкретные ресурсы.

### Механизм обработки запросов

При каждом обращении проверяется токен аутентификации пользователя. Затем определяется роль текущего пользователя и сопоставляется с правами доступа. Запросы выполняются успешно, если пользователь обладает необходимой ролью и соответствующими правами.

Если у пользователя нет достаточной роли или полномочий, возвращаются коды ошибок 401 ("Unauthorized") или 403 ("Forbidden").

### Минимальная демонстрационная база данных

Таблицы заполняются минимальным объемом данных для показательной демонстрации работы системы.

### Рекомендуемая технология

Используются Python/Django Rest Framework и PostgreSQL.
